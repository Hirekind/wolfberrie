Settings Redesign

As is typical with startups, the settings pages of Coinbase had never really been planned properly and new settings had been added without much concern for usability. I was asked to take the settings pages, and redesign them from scratch.

User testing

To figure out the best navigational structure for the settings I conducted thorough user tests. Below you can see examples of some of the click tests I ran with the Usabilityhub tools.

You are trying to change your password. Which of these links would you click?

You want to change your time zone and currency. Where would you click?

You want to set up 2-factor authentication. Where do you click?

Old navigation structure

Below is how the navigational structure looked like before the changes were implemented.

New navigation structure

When planning the new navigation I used the results of the user testing as the guiding information. The page "Personal details" was completely removed and the content was moved to the "User profile" page.

Old Page

The old page had the settings organized in a relatively random manner. The old settings pages generated a significant amount of support requests every week.

New Page

The new page is more organized with the most important settings in the beginning. All the settings have been placed on their specific places based on the data gathered in the user tests.

Results

After implementation of the new settings pages we were able to eliminate multiple support requests altogether. We also reduced the amount of weekly requests on most of the other settings related issues.
